2 Tokenization: Morgan Fingerprint (Radius 0)

How atoms become tokens

Each heavy atom is fingerprinted with a Morgan fingerprint of radius 0 — meaning each atom is encoded by its own elemental type, hybridization, and aromaticity. No neighborhood information.

info = {}
AllChem.GetMorganFingerprint(mol, radius=0, bitInfo=info,
    includeRedundantEnvironments=True)
# info = {bit_id: [(atom_idx, radius), ...], ...}

For ethanol (CCO):

Atom IndexElementMorgan BitVocabulary TokenToken ID
0C (sp³, aliphatic)bit_42C_sp3_aliphatic17
1C (sp³, bonded to O)bit_58C_sp3_adjO23
2O (sp³, hydroxyl)bit_107O_sp3_hydroxyl89

* Token IDs are illustrative — the actual vocabulary has 207 atom-environment tokens + 4 specials = 211 total. Special tokens: [PAD]=0, [CLS]=1, [SEP]=2, [MASK]=208, [UNK]=210

Why radius 0? Larger radii would encode graph structure INTO the token identity. The discriminator could then trivially detect replaced tokens by pattern-matching atom neighborhoods. Radius 0 forces the model to learn structure from relative positions, making RTD genuinely hard.

3 Molecular Graph → Distance Matrix

Bond distances become the positional encoding
dist_mat = Chem.GetDistanceMatrix(mol) # shortest path in bonds

Distance matrix for ethanol:

C₁C₂O C₁012 C₂101 O210

Diagonal = 0 (self).
Off-diagonal = bond distance.

C₁↔C₂ = 1 (directly bonded)
C₂↔O = 1 (directly bonded)
C₁↔O = 2 (two bonds away)
Disconnected atoms → 1e8 (set to -1 then shifted to 0)
Key insight: The distance matrix IS the molecule's positional encoding. Unlike NLP (where position = sequence order), molecules have graph structure — atoms are related by bond paths, not linear order. These distances feed directly into disentangled attention as relative positions.

Tokens → dense vectors. Position info comes ONLY from relative attention.

🔑 Critical design decision: MolE-RTD uses NO absolute position embeddings. Unlike BERT (which adds position[i] to token[i]), MolE-RTD relies entirely on graph-derived relative positions fed through disentangled attention. The molecular graph IS the position encoding.

6 Relative Position Embedding Lookup

Each bond-distance maps to a learned 64-dim vector

The relative position embedding table has shape [1024, 64] = [2 × max_relative_positions, head_size] where max_relative_positions = 512.

Position index rel_pos (the bond distance) maps to embedding row:

row = rel_pos + max_relative_positions = rel_pos + 512
DistanceRow Index→Embedding (first 4 of 64 dims)
0 (self)512→[0.00, 0.00, 0.00, 0.00, …]
1 (bonded)513→[0.03, -0.01, 0.02, 0.01, …]
2 (two bonds)514→[0.01, 0.02, -0.01, 0.03, …]
3515→[-0.01, 0.03, 0.01, 0.02, …]

Position 0 (self-distance) is explicitly zeroed during training setup with rel_embeddings.weight[0].fill_(0).

7 Disentangled Attention (The Heart of DeBERTa)

Three attention terms: Content↔Content + Content→Position + Position→Content

Standard self-attention: softmax(QK^T / √d) × V

Disentangled attention adds two cross terms:

Attention = softmax( QcKcT + QcKrT + QrKcT ) × V
TermNameWhat it means for molecules
QcKcT Content→Content "How much does atom i's identity attend to atom j's identity?"
e.g., "Carbons attend to nearby oxygens"
QcKrT Content→Position (c2p) "Given atom i's identity, how much does it care about atoms at distance d?"
e.g., "An sp² carbon cares about atoms 1 bond away (direct neighbors)"
QrKcT Position→Content (p2c) "Given distance d, how much should atoms at that distance attend to atom j's identity?"
e.g., "At distance=1 (bonded), pay attention; at distance=5, ignore"

9 RTD Pretraining (Generator → Discriminator)

Replaced Token Detection — every token gets a training signal

Generator (3 layers, ~7M params)

Small model that corrupts the input:

  1. 15% of tokens are masked (never [CLS])
  2. Of masked: 80% → [MASK], 10% → random token, 10% → unchanged
  3. Generator predicts original token at masked positions
  4. Generator's best guess replaces the masked tokens
  5. Loss: CrossEntropy(gen_logits, mlm_labels) on masked positions only

Discriminator (12 layers, ~89M params)

Full model detects replaced tokens:

  1. Takes generator's corrupted sequence as input
  2. For EVERY token: predict "original" (0) or "replaced" (1)
  3. Loss: BCE(disc_logits, disc_labels) on ALL tokens
  4. Label = 1 only if generator replaced token AND got it wrong
  5. This is the model we keep for downstream tasks!

Concrete example with ethanol:

Step[CLS]C₁C₂O
Original tokens1172389
After masking (15%)1[MASK]2389
Generator predicts117 ✓2389
Discriminator input1172389
Discriminator labels00 (correct!)00

In this case the generator got it right (predicted 17), so the discriminator label is 0 (original). The discriminator must learn to distinguish "genuine atom in this context" from "generator inserted something wrong."

Loss function:

Ltotal = Lgen + 50 × Ldisc

λ=50 weights the discriminator heavily — it's the model we actually keep.

10 GDES: Gradient-Disentangled Embedding Sharing

How generator and discriminator share embeddings without gradient interference
Wdisc = Wgen.detach() + bdisc
ComponentUpdated byDescription
Wgen Generator MLM loss 211 × 768 embedding matrix. Generator gradients flow normally.
bdisc Discriminator RTD loss 211 × 768 bias, initialized to zeros. Only discriminator-specific delta.
Why GDES? Without it, the discriminator's gradients would update Wgen too, coupling the two models. With GDES, the generator learns good token embeddings from MLM, and the discriminator learns subtle deviations (bdisc) from those embeddings for RTD. Both share the same semantic space without interfering.

🎯 Key Design Decisions (Why Each Choice Matters)

DecisionWhat it meansWhy
RTD over MLM Binary detection on ALL tokens vs. predicting only masked 15% Every token gets a gradient → richer representations per step
Radius 0 tokens Atoms encoded by type only, not neighborhood Otherwise RTD is trivial — "this neighborhood doesn't match"
No absolute positions position_biased_input=False Molecules don't have a "first atom" — graph structure IS position
Bond distances = rel pos Shortest path in molecular graph → attention bias Natural inductive bias for molecular modeling
GDES detach + bias sharing Shared semantic space without gradient interference
λrtd = 50 Discriminator loss weighted 50× over generator Discriminator is the model we keep; generator is auxiliary
Frozen encoder + XGBoost No fine-tuning on small downstream datasets 89M params on 813 samples = guaranteed overfitting
Pre-tokenized shards .pt files with tokens + edge_index baked in 2× throughput, avoids CPU tokenization during training
